# Verilog RTL 设计方法学

## 硬件思维铁律（Agent 行为约束）

以下约束为 AI 在解答数字 IC 设计问题时的强制性思维框架。违反任何一条将导致错误的软件化结论。

### 底层视角
所有问题拆解为：**门电路、触发器、寄存器、组合逻辑、时序逻辑、总线、时钟域、同步/异步**。不默认运行在通用 CPU/OS 上。

### 执行模型
- 循环 → **状态机、流水线级、并行硬件阵列、时序轮转**
- 条件分支 → **多路选择器 MUX、与/或/非门、译码器**
- 数据存储 → **寄存器、寄存器堆、SRAM/ROM、锁存器、FIFO**（非"变量/数组/缓存"）
- 运算 → **加法器、乘法器、移位器、比较器、运算单元阵列**

### 数据视角
所有数据以 **bit 位宽、位拼接、位截取、码制（原码/补码）、电平（0/1/Z/X）** 为基础分析。关注位宽溢出、总线匹配。

### 时序核心
任何功能必须考虑：**CLK、RST、建立时间 Tsu、保持时间 Th、时序收敛、亚稳态、跨时钟域 CDC、组合逻辑毛刺**。禁用软件延时/休眠。

### 架构视角
优先：**流水线、并行阵列、时分复用、串行/并行总线、模块级互联、Valid/Ready 握手**。禁用：函数调用、API、类/对象、线程/进程。

### 性能度量
分析指标：**最高工作频率 Fmax、吞吐率、延迟 Latency、硬件资源（门数、FF 数、RAM 大小）、功耗**。不使用软件运行时间/内存占用。

## 核心工作流

### 设计新模块

1. **定义接口契约**：端口方向/位宽、操作模式、时序（延迟/吞吐）、数据格式、边界情况（零/溢出/下溢）
2. **选定流水线模板**：`IDLE → S0 → ... → SN → IDLE`，含 i_ready/o_valid 握手
3. **参数化一切**：`parameter` 声明位宽，`localparam` 派生常量，绝不硬编码
4. **按流水级组织**：每级 = 组合逻辑块（`always @(*)`，阻塞 `=`）+ 时序寄存器块（`always @(posedge clk)`，非阻塞 `<=`）
5. **末级直出最终格式**：不在流水线之后加独立后处理转换

### 硬件资源复用

多模式设计中，不同模式的并行度与数据位宽常呈反比。若按"最大并行度 × 最大位宽"声明所有寄存器/运算器，低精度模式下大量高位 bit 闲置，造成面积浪费。

**核心方法：以物理资源上限为约束，通过模式感知的打包/解包逻辑，在不同模式间动态复用同一组物理资源。**

六项原则：

1. **先分析再设计**：枚举所有模式下的并行度 × 位宽需求矩阵，识别各子系统的峰值需求与闲置区间
2. **物理资源定上限，逻辑资源做适配**：寄存器/运算器的物理规格按各模式的**合理交集**定义（非最大值的简单笛卡尔积）
3. **打包在出口，解包在入口**：模块接口输出紧凑格式节省跨级寄存器；模块内部展开全格式便于逐通道处理
4. **参数化声明，不硬编码**：通过 `parameter/localparam` 定义可配置的物理槽位/位宽/数量，便于裁剪和扩展
5. **旁路优于虚耗**：能被简单位移/加法替代的运算，不做完整运算器实例化
6. **对等复用优先**：两个信号的位宽和计算模式完全对等时，通过索引偏移复用同一组硬件资源

> 详见 [REFERENCE.md §1.5 硬件资源复用方法论](REFERENCE.md) — 含乘法器阵列复用、寄存器位拆分存储的完整案例与分析。

### 编码规则速查

| 规则 | 正确 | 错误 |
|------|------|------|
| 组合/时序分离 | `always @(*)` 用 `=`，`always @(posedge clk)` 用 `<=` | 同一块内混用 |
| 防锁存器 | 组合块顶部设默认值 | 条件分支遗漏赋值 |
| 总线 | `[WIDTH-1:0]` | `[0:WIDTH-1]` |
| 有符号运算 | 显式 `signed`、`$signed()` | 依赖默认无符号 |
| 无关位 | `casez` | `casex` |
| 时钟门控 | `if (enable)` 无 else | `if-else` 含显式保持 |
| 累加有符号值 | XOR 补码 → 单加法器树 | `if/else +/-` → 双加法器树 |
| 实例化 | 命名连接，端口上无逻辑 | 端口上直接运算 |

### 数值系统

- 每次运算审计位宽：加法需 `max(a,b)+1`，乘法需 `a+b`
- 无符号减法前判断下溢可能，加显式保护
- 补码取反必须在原始位宽下操作（先取反再扩展，非反之）
- 可能为负的值做无符号范围比较前，先检查 MSB 符号位
- 浮点流水线中跟踪每级指数偏移残余量

### 调试（4 阶段渐进）

1. **定位** — `--list-signals` 发现信号层级
2. **广角快照** — `--watch` 在故障时刻获取全系统状态
3. **聚焦追踪** — `--watch <信号> --trigger <沿> -n N` 或 `--time T1-T2`
4. **迭代深入** — 根据发现缩窄窗口，钻探可疑子模块
